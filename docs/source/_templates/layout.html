{# layout.html =========== PyData Sphinx Theme layout override. Compatible: PyData 0.7 - 0.17+ (all versions). Strategy: render → stage → relocate ------------------------------------- Jinja's super() is atomic — no parent template can inject content at an interior position inside a child block. The only correct non-extension solution is: 1. Render the Jinja component here (Jinja runs fully, all sub-includes resolve: shell_py_example.html, shell_interactive.html, etc.) 2. Place rendered HTML into a hidden staging `div` with a known id. This div sits outside the article DOM. 3. Emit a `script` that runs at DOMContentLoaded, finds the CSS selector target inside the article, and moves the staging div's contents to that exact position using insertAdjacentHTML — the standard DOM API, no libraries. Why this is correct and stable -------------------------------- - Jinja rendering happens at build time (server side) — all {% include %} calls resolve through Sphinx's own Jinja loader. - DOM relocation happens at load time (client side) — only standard DOM APIs used (insertAdjacentHTML, getElementById). - No dependency on PyData internals. No version-specific blocks. - CSS selectors in conf.py are stable sphinx-gallery classes names. Safety guards ------------- Guard 1 — Jinja inheritance: index.html overrides docs_main entirely → this block is unreachable on the main page. Architectural, not conditional. Guard 2 — page_components dict lookup: Only pages explicitly listed in conf.py receive components. Unknown pagenames produce an empty _cfg → nothing emitted. Guard 3 — JS selector check: If target element not found on page → logs to console, drops silently. Page always loads correctly. Guard 4 — ignore missing: If template file does not exist → silently skipped. Build always completes. Adding a new page ----------------- Edit conf.py html_context["page_components"] only. Do not edit this file. #} {% extends "!layout.html" %} {% block docs_main %} {# ------------------------------------------------------------------ #} {# Resolve config for this page from conf.py html_context #} {# ------------------------------------------------------------------ #} {%- if page_components is defined %} {%- set _cfg = page_components.get(pagename, {}) %} {%- else %} {%- set _cfg = {} %} {%- endif %} {%- set _template = _cfg.get("template", "") %} {%- set _selector = _cfg.get("selector", "") %} {%- set _position = _cfg.get("position", "beforebegin") %} {# ------------------------------------------------------------------ #} {# Original page content — untouched #} {# ------------------------------------------------------------------ #} {{- super() }} {# ------------------------------------------------------------------ #} {# Stage + relocate — only when this page has a component configured #} {# ------------------------------------------------------------------ #} {%- if _template and _selector %} {#- Render the Jinja component fully at build time. #} {#- All sub-includes resolve here via Sphinx's Jinja loader. #} {#- The staging div is hidden — user never sees it at this position.#} {#- Unique staging id derived from pagename — valid HTML id, no loop needed #}
